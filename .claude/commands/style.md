지정된 컴포넌트나 페이지의 스타일을 개선합니다.

## 입력
$ARGUMENTS — 대상 파일 경로 또는 컴포넌트명 (예: `pages/Dashboard`, `components/common/Pagination`)

## 규칙
- Tailwind CSS 유틸리티 클래스만 사용 (인라인 style, CSS 파일 추가 금지)
- 반응형 대응: sm / md / lg 브레이크포인트 활용
- hover, focus 상태 포함
- 다크모드 고려 불필요 (현재 라이트 모드만)
- 간격, 색상, 폰트 사이즈 등 일관성 유지
- 기존 레이아웃 구조를 깨지 않는 범위에서 개선
